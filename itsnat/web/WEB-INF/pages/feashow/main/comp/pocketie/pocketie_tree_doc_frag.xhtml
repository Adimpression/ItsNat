This example shows a free tree based component.

There are several significative differences compared with the "normal" example:

  1. Handle, icon and label are covered with links. These links are used to receive mouse events to expand/collapse and select tree nodes as a workaround to surpass Pocket IE/Symphony/UCWEB limitations (see "Free Combo Box" explanation for more info).

  2. The component is configured as setSelectionUsesKeyboard(false) because all of them are mobile browsers.

This example uses the same FreeTreeDecorator class used in any other tree example (including the main tree of the Feature Showcase), because this class detects an HTML tree ready to work with links as wrappers.
